## Module 1 - The Nature and Purpose of Intelligent Systems

### TLDR
Intelligent systems are built on machine learning (ML), which has four core learning paradigms: **supervised** (task-driven, labelled data), **unsupervised** (data-driven, unlabelled), **semi-supervised** (hybrid), and **reinforcement** (environment-driven, reward/penalty). This module introduces these foundations alongside real-world applications (cybersecurity, healthcare, IoT, NLP) and the ethical considerations that must guide AI deployment â€” including fairness, bias, and differential privacy. ISY503 focuses primarily on supervised learning and models, with reinforcement learning for AI agents.

### Introduction
Intelligent systems have been studied in the following areas: speech recognition, computer vision and natural language processing. This subject focuses primarily on computer vision and natural language processing, as the field of speech recognition is nested in these two main areas. It also covers the basics of machine learning (ML), as intelligent systems are based on ML.

This module is an introduction to intelligent systems. It will provide you with an understanding of the reasoning behind intelligent systems and their applications. ## Module 2 - Introduction to Machine Learning

### TLDR

- **Supervised learning** maps labelled input **features (X)** to output **targets (Y)** â€” classification produces a discrete class label; regression produces a continuous number.
- Data must be split into **training** (model learns from this) and **testing** (true generalisation measure) sets before fitting any model; mixing them causes optimistic bias (**data leakage**).
- Core evaluation metrics all derive from the **confusion matrix** (TP, TN, FP, FN): accuracy (balanced data), precision (FP-sensitive), recall (FN-sensitive), F1 (imbalanced data), and AUC (threshold-independent comparison).
- **Overfitting** â€” great training accuracy, poor test accuracy â€” is countered by **feature selection** (remove irrelevant/redundant features via filter/wrapper/embedded methods) and regularisation.
- **Categorical features** must be encoded numerically before most ML algorithms can process them; key strategies are **Label Encoding** (integers) and **One-Hot Encoding** (binary columns), with **target encoding** (regularised) best for high-cardinality features.
- For **reinforcement learning**, rewards replace labelled targets â€” the reward function quality determines learning quality; for **unsupervised learning** (clustering), internal metrics (Silhouette, Davies-Bouldin) evaluate structure quality without ground-truth labels.

### Introduction
In this Module, you will learn about the basics of machine learning (ML). Intelligent systems are based on computersâ€™ or machinesâ€™ perceptions about the world. Thus, ML plays an important role in intelligent systems. In this Module, we focus on the supervised learning part of ML. Supervised learning requires labelled data to make a relation between the input variables and output variables. 
- **The inputs are called features** and the **outputs are called targets**. 
- The algorithms that find the **relations between features and targets are called classifiers**. 
Based on the relation they find, they can classify or predict the outcome of data they have not seen previously. There are numerous classifiers, each of which has different specifications. Based on the given labelled data, they can make mistakes when predicting the outcome of a new data. To evaluate their performance, some data can be given to them as a training dataset so they can find the relation. Afterwards, their performance will be measured by administering a testing dataset. This Module also discusses how to split a given dataset into training and testing datasets. The performance of classifiers on training and testing datasets are measured by ML metrics. You will examine each metric in further detail in this Module, which will help you to better understand your case study for your first assessment.
